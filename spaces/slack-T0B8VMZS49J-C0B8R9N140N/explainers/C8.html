C8 · 多头注意力改善表达

*C8 钩子：为什么 Transformer 不只做一个 attention，而要做 8 个？*

单头 attention 已经能看全句；多头 attention 的主张更细：不是“多看几遍”，而是让模型在*不同表示子空间*、*不同位置关系*上同时看。

*关键直觉：一个头像一支探照灯，多头像一组分工不同的探照灯。*

同一个 token 生成 query/key/value 后，会被投影到多个不同子空间。每个 head 可以学一种不同的“相关性”：比如语法依赖、指代、局部搭配、远距离对应。

Fig.2 右图直接展示了 multi-head attention：多个 attention 层并行运行，最后 concat 再投影。

*论文给出的机制非常直接：先分头投影，再并行 attention，再拼回去。*

> Multi-head attention allows the model to jointly attend to information from different representation subspaces at different positions.

也就是说，多头不是输出多个答案，而是把多个子空间里的注意力结果合成一个表示。

*为什么单头不够？论文点名了一个问题：平均会损失分辨率。*

> With a single attention head, averaging inhibits this.

单头 attention 最后是对 values 做加权平均；如果一个位置同时需要吸收几类不同信息，单一平均可能把结构揉在一起。多头用分开的子空间缓解这个问题。

*常见误解：多头不是简单把参数量暴力放大。*

论文里 base model 用 `h = 8`，每个 head 的维度是 `d_k = d_v = d_model / h = 64`。因为每个头变窄，总计算量大致接近一个全维单头 attention。

*收尾：C8 的核心不是“attention 更强”，而是“attention 被结构化成可并行分工”。*

多头让 Transformer 在同一层里同时建立多种位置-表示关系：不同 head 看不同子空间，不同位置贡献不同信息，最后合成一个更丰富的 token 表示。
