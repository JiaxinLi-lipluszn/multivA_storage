C4 · 位置编码补偿无序结构

*C4 钩子：Transformer 不是“天然懂顺序”的。*

它把循环和卷积都拿掉了，所以 attention 本身主要是在比较 token 表示之间的关系；如果不给它位置信号，模型没有内置的“第几个词”概念。

*关键直觉：同一个词，出现在不同位置，词嵌入还是同一个。*

`bank` 在第 2 个位置和第 20 个位置，token embedding 本身不会变。RNN 用时间步带顺序，CNN 用局部窗口带顺序；Transformer 需要额外把位置坐标塞进去。

图1里 Positional Encoding 被加在 encoder/decoder 输入嵌入的底部，说明顺序信息是在进入注意力层前注入的。

*论文机制：位置编码和词嵌入直接相加。*

> “we add ‘positional encodings’ to the input embeddings”

原因是二者维度相同：`d_model = 512`。所以每个输入位置实际进入模型的是：

`token embedding + positional encoding`

*它用的不是一个位置编号，而是一组正弦/余弦坐标。*

论文给出：
`PE(pos, 2i) = sin(pos / 10000^(2i/d_model))`
`PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))`

意思是：每个位置被编码成一个 512 维向量，不同维度对应不同频率的波。

*常见误解：位置编码不是替代 attention，而是让 attention 有顺序可用。*

Self-attention 负责“谁和谁相关”；positional encoding 负责告诉它“这些 token 在序列中的相对/绝对位置”。没有后者，模型很难区分 `A before B` 和 `B before A` 这种顺序差异。

*收尾：C4 的核心是补偿。*

Transformer 的并行性来自去掉 recurrence/convolution；代价是结构本身不再携带序列顺序。位置编码就是这笔账的补偿：把顺序信息加回输入，让纯 attention 架构仍能处理序列。
