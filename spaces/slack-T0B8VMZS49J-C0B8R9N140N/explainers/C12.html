C12 · 自注意力路径更短

*C12: 自注意力把“远距离依赖”的路程压短了。*

在 RNN 里，位置 `i` 的信息要传到很远的 `j`，通常要沿时间步一格格走；在 Transformer 的 self-attention 层里，`i` 和 `j` 可以在同一层里直接发生关系。

> In the Transformer this is reduced to a constant number of operations

关键直觉：*序列距离不再等于计算路径长度*。

self-attention 不是“从左到右传话”，而是每个位置都拿自己的 `query` 去看同一序列里所有位置的 `key/value`。所以第 3 个词和第 300 个词之间，不需要经过 297 个中间 hidden states。

Scaled dot-product attention 的核心是让每个 query 同时和所有 keys 比较，因此远近位置都能在一次 attention 中相连。

论文把这个优势直接量化在 Table 1：

> Self-Attention: Maximum Path Length O(1)
> Recurrent: Maximum Path Length O(n)
> Convolutional: Maximum Path Length O(log_k(n))

这里的 `Maximum Path Length` 可以理解成：两个任意位置的信息要互相影响，最坏要经过多少层/步计算。self-attention 是常数级。

为什么这重要？论文的说法是，路径越长，远距离依赖越难学。

> This makes it more difficult to learn dependencies between distant positions

所以 C12 不是单纯说“attention 看得远”，而是说：*远距离依赖在计算图里的通信路径更短*。这会影响优化、梯度传播和依赖建模。

一个容易误解的点：`O(1)` 路径长度不等于“免费”。

self-attention 的每层复杂度是 `O(n^2 · d)`，因为所有位置两两打分；RNN 是 `O(n · d^2)`，路径长但不是二次 token-token 交互。

所以这条主张讲的是 *信息路径长度*，不是说 self-attention 在所有维度上都更便宜。

收尾：Transformer 的一个核心设计胜负手是，把“远处 token 要经过很多步才能相互影响”改成“任意两个位置一层内可直接交互”。

这也是为什么论文在介绍 Transformer 时反复强调：它不是只换了一个 attention 模块，而是用 self-attention 取代了序列对齐的 recurrence/convolution。
