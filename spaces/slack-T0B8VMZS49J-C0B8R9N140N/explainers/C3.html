C3 · 自注意力承担全局依赖建模

*C3 主张：Transformer 里的长距离依赖，主要交给自注意力来建模。*

钩子是：它不是把 RNN 里的 recurrence 换个名字，而是把“位置之间怎么互相取信息”变成模型的核心操作。

> relying entirely on an attention mechanism to draw global dependencies between input and output

*关键直觉：距离不再等于计算路径。*

在 RNN 里，两个很远的 token 要通过很多时间步传递信息；在卷积里，也要靠多层或更大感受野逐步覆盖。自注意力的思路更直接：一个位置可以一次性对整段序列里的位置分配权重。

Scaled dot-product attention 把一个 query 变成对所有 value 的加权汇总；multi-head 让模型在不同子空间、不同位置关系上并行地做这件事。

*论文里的机制说得很直接。*

Encoder 里，每个位置都能看同一序列的所有位置；Decoder 里，每个位置能看已经生成的输出位置；encoder-decoder attention 则让输出位置看输入序列。

> Each position in the encoder can attend to all positions in the previous layer of the encoder.

> This allows every position in the decoder to attend over all positions in the input sequence.

*证据点：论文把“任意两点相连的路径长度”当成核心优势。*

> In the Transformer this is reduced to a constant number of operations

表 1 里也把 `Self-Attention` 的 maximum path length 标成 `O(1)`，而 `Recurrent` 是 `O(n)`，卷积是 `O(log_k(n))`。这就是“全局依赖更容易直接建模”的结构性理由。

*常见误解：自注意力不是自动懂顺序。*

因为 Transformer 没有 recurrence 和 convolution，它必须额外注入位置信息。

> Since our model contains no recurrence and no convolution, in order for the model to make use of the order of the sequence, we must inject some information about the relative or absolute position of the tokens in the sequence.

*收尾：C3 的准确说法不是“注意力解决一切”，而是：*

Transformer 把跨位置、长距离的信息路由主要交给自注意力；feed-forward 层做逐位置变换，positional encoding 补顺序信息，multi-head attention 缓解单头加权平均带来的表达损失。
