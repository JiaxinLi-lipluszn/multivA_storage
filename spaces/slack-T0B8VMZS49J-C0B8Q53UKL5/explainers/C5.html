C5 · 正弦位置编码可表达相对位移

*C5 钩子：Transformer 的位置编码不是只告诉模型“这是第几个 token”。*

它还有一个很漂亮的性质：对固定偏移 `k`，位置 `pos+k` 的编码可以由 `pos` 的编码线性表示。

也就是说：模型有机会用很简单的线性变换学到“相隔 3 个词”“往前 7 个词”这类相对位置关系。

*为什么需要这件事？*

Transformer 没有 RNN 的时间步，也没有卷积窗口；如果不额外注入位置，self-attention 本身看不出 token 顺序。

> Since our model contains no recurrence and no convolution, in order for the model to make use of the order of the sequence, we must inject some information about the relative or absolute position of the tokens in the sequence.

*论文怎么做：把位置写成一组不同频率的正弦/余弦。*

> In this work, we use sine and cosine functions of different frequencies:

`PE(pos, 2i) = sin(pos / 10000^(2i/d_model))`
`PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))`

每个 `i` 对应一个频率；一对维度就是同一个频率下的 `sin` 和 `cos`。

同一频率下，固定偏移 k 等价于对 sin/cos 二维子空间做一个固定线性变换。

*关键机制：三角恒等式把“平移”变成“线性”。*

对某个频率 `λ`：
`sin((pos+k)/λ) = sin(pos/λ)cos(k/λ) + cos(pos/λ)sin(k/λ)`
`cos((pos+k)/λ) = cos(pos/λ)cos(k/λ) - sin(pos/λ)sin(k/λ)`

注意：`k` 固定时，`cos(k/λ)` 和 `sin(k/λ)` 都是常数。所以 `PE(pos+k)` 可以由 `PE(pos)` 线性算出来。

*常见误解：这不等于模型“自动懂语法距离”。*

正弦位置编码只是把相对位移做成一种容易被线性层/注意力层利用的表示；真正是否学会“向前看 1 个词”“隔几个 token 对齐”，还要靠训练数据和 attention/FFN 参数去学。

它提供的是一种 inductive bias，不是硬编码规则。

*收尾：C5 的核心一句话。*

Transformer 用不同频率的 `sin/cos` 给每个位置编码；因为固定偏移在每个 `sin/cos` 二维子空间里都是线性变换，所以模型不只拿到了绝对位置，也拿到了更容易学习相对位置关系的坐标系。
