C2 · 编码器解码器可由注意力堆栈实现

*C2 钩子：Transformer 不是“在 RNN 上加 attention”*
它保留 encoder-decoder 的任务形式，但把主体骨架换掉：encoder 和 decoder 都由注意力层与逐位置前馈网络堆叠起来。
> “stacked self-attention and point-wise, fully connected layers”

Figure 1 直接展示了这条主张：encoder/decoder 都是重复堆叠的 attention + FFN 结构。

*关键直觉：序列建模不一定要按时间步递推*
RNN 的默认思路是 `h_t` 依赖 `h_{t-1}`，所以训练时位置之间天然串行。
Transformer 的换法是：每一层里，让每个 token 通过 self-attention 直接看同一序列里的其他位置，再用 FFN 做逐位置变换。

*机制：encoder 和 decoder 的堆栈不完全一样*
• Encoder 每层：multi-head self-attention + position-wise FFN
• Decoder 每层：masked self-attention + encoder-decoder attention + position-wise FFN
• 论文实现里两边都是 `N = 6` 层
Decoder 多出来的 encoder-decoder attention，让输出端每个位置都能对输入端所有位置做注意力。

*为什么说“足以”？论文给的是翻译任务上的实证结果*
• WMT 2014 English-to-German：`28.4 BLEU`
• WMT 2014 English-to-French：`41.8 BLEU`
这些结果支撑的是一个结构性结论：不用 recurrence / convolution，纯 attention 堆栈也能完成高质量 seq2seq 翻译。

*常见误解：decoder 不是可以偷看未来*
Decoder 的 self-attention 是 masked 的：位置 `i` 只能依赖已生成的更早位置。
它可以看完整输入，是通过 encoder-decoder attention；但它不能看未来输出 token。这样才保留自回归生成。

*收尾：C2 的重点是“骨架替换”*
这篇论文不是只提出一个 attention 小模块，而是说：encoder-decoder 机器翻译系统的主体可以由 multi-head self-attention + position-wise FFN 堆栈承担。
值得追问的下一步是：既然没有 recurrence / convolution，序列顺序从哪里来？论文的答案是额外加入 positional encoding。
